## Anthropic's own framing

> "A modest but tangible improvement on its predecessor."

So: not a leap, but the improvements compound with Dynamic Workflows.

## Benchmark deltas (vs Opus 4.7)

| Dimension | 4.7 | 4.8 |
|---|---|---|
| Agentic coding | 64.3% | **69.2%** |
| Knowledge-work score | 1753 | **1890** |
| Agentic computer use | improved | improved (no exact %) |
| Multidisciplinary reasoning | improved | improved (no exact %) |

## What materially helps Dynamic Workflows

### 1. ~4× less likely to leave code flaws unremarked

The model flags problems it sees in code it just generated, instead of
silently shipping a buggy patch. In a 1,000-agent workflow where each
agent's output is reviewed by another, this lowers the false-clean rate
*per pair*, which compounds across the run. This is probably the single
biggest reason 4.8 + Dynamic Workflows is more useful than 4.7 + the same.

### 2. Mid-conversation `role: "system"` messages

Opus 4.8 accepts system messages **immediately after a user turn** in the
messages array. You can append updated instructions partway through a
long-running conversation without restating the whole system prompt.

For workflows specifically: each sub-agent run can append refined
guidance from the orchestrator without paying input-token cost on the
full prompt every time. Especially relevant for the **adversarial-review
pattern** (one agent finds, another refutes) where the refuter needs
sharpened criteria.

### 3. Fast mode — 2.5× speed, 3× cheaper

Material for workflows because:
- The Haiku scouts (cheap breadth) in Video 3's "Haiku finds, Opus
  synthesizes" pattern get even cheaper
- Long-running overnight workflows (Bun-style optimization pass) finish
  sooner
- Per-run token bills come down even at constant agent counts

### 4. Long context + high output

- **1M token context** (unchanged from 4.7)
- **128K max output** (unchanged)

Context size matters less when workflows move intermediate state into
script variables — but it still helps for the synthesis-step Opus call
that writes the final answer.

## Knowledge cutoff

January 2026 — same as 4.7. Workflows that need post-Jan-2026 facts
still need MCP/web tools or a web-search agent stage.

## The cynical read (Simon Willison)

> "Anthropic calling Opus 4.8 'a modest but tangible improvement'
> matches my sense of it. The interesting product story this week is
> dynamic workflows, not the model bump."

The model bump matters because *workflows multiply per-agent quality* —
but Dynamic Workflows is the headline. Opus 4.8 is the substrate that
makes the headline cheaper and more reliable.
